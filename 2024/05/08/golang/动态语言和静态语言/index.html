# 静态语言

概念：静态语言在编译时执行类型检查，并在编译期完成类型绑定。这意味着变量在运行之前就已经绑定了特定的数据类型，编译器在编译期就能发现类型错误。例如 C、C++、Java、Go 等。

优点：

  • 编译时的类型检查可以减少运行时的类型错误
  • 编译器能够进行更多的代码优化，运行时性能较好
  • 代码更容易被 IDE 和编译器理解，更利于代码分析和重构

缺点:

  • 编码时需要指定变量的类型，写代码的灵活性略低于动态类型
  • 编译过程相对耗时较长，开发周期不如动态语言来得快

# 动态语言

概念：动态语言在运行时进行类型检查和绑定，即变量仅在被赋值时才绑定特定的类型。任何对象都可以绑定到任何变量。例如 Python、Ruby、JavaScript 等。

优点:

  • 变量无需提前声明类型，写代码灵活性很高
  • 可以快速迭代，开发周期相对短
  • 动态类型语言函数通常可以使用参数个数不等的方式被调用

缺点:

  • 运行时的类型检查会降低执行效率
  • 相比静态语言，缺乏编译时的类型检查，更难发现潜在的类型错误
  • 动态类型代码行为很难被分析工具预先推断，不太利于 IDE 开发

# 静态语言和动态语言变量的区别

# 静态语言

  • 必须在使用前声明变量类型
  • 类型确定后不能改变
  • 先声明后使用

# 动态语言

  • 无需事先声明变量类型，变量可以在被赋值时自动绑定类型
  • 变量的类型是动态的，可以在运行时改变
  • 变量内存通常在运行时动态分配，访问速度相对较慢
